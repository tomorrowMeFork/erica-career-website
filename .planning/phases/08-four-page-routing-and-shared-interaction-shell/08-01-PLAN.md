# Plan 08-01: Four-Page Routing and Shared Interaction Shell

**Phase:** 08 — Four-Page Routing and Shared Interaction Shell  
**Status:** Ready for execution  
**Requirements:** IA-02, UXR-02  
**Created:** 2026-05-04

## Goal

Replace the Phase 5 single dashboard with four App Router pages plus a shared Korean-first navigation shell, without changing APIs or existing component interfaces.

## Route Map

| Route | File | Purpose |
|---|---|---|
| `/` | `app/page.tsx` | Home / service orientation |
| `/explore` | `app/explore/page.tsx` | Career information explore shell |
| `/source/[id]` | `app/source/[id]/page.tsx` | Source verification shell |
| `/consultation` | `app/consultation/page.tsx` | Career consultation / chat |
| Shared | `components/shell/app-shell.tsx` | Persistent header + navigation |
| Root | `app/layout.tsx` | Wrap `{children}` with `AppShell` |

## Task Breakdown

### Task 1: Shared Shell Component

**Output:** `components/shell/app-shell.tsx`  
**Requirements:** IA-02, UXR-02

- Client component (uses `usePathname()` from `next/navigation`).
- Navigation labels from Phase 7 flow-language contract:
  - 홈 -> `/`
  - 커리어 정보 탐색 -> `/explore`
  - 출처 확인 -> `/source/*` (active for any `/source/` path)
  - 커리어 상담 -> `/consultation`
- Active state: `aria-current="page"` on current link.
- Desktop: persistent header nav bar.
- Mobile: compact bottom nav with icons/labels.
- Wrap `{children}` in the shell body.

### Task 2: Root Layout Update

**Output:** Edit `app/layout.tsx`  
**Requirements:** IA-02

- Import `AppShell`.
- Wrap `{children}` with `<AppShell>{children}</AppShell>`.
- Keep `<html lang="ko">`, metadata, Pretendard font class, and `globals.css` import.

### Task 3: Home Page

**Output:** Edit `app/page.tsx`  
**Requirements:** IA-02, UXR-02

- Remove `StudentDashboard` import.
- Render home orientation: service purpose, source-grounding summary, primary CTA link to `/consultation`, secondary link to `/explore`.
- Include `DisclaimerNotice`.
- Korean-first copy from Phase 7 language contract.

### Task 4: Explore Page

**Output:** `app/explore/page.tsx`  
**Requirements:** IA-02

- Client component.
- Move recommendation/listing state from `StudentDashboard`:
  - `sessionKey`, `recommendations`, `recommendationResponse`, `activeFilter`, `preferenceState`
  - `refreshRecommendations`, `handleSave`, `handleUpdate`, `handleClearPreferences`, `handleRead`
- Render `ListingPanel` (existing import path unchanged).
- Placeholder for Phase 9 browse/filter content.

### Task 5: Source Verification Page Shell

**Output:** `app/source/[id]/page.tsx`  
**Requirements:** IA-02

- Placeholder shell for Phase 9 implementation.
- Korean-first framing: "이 정보가 어떤 원문과 근거에서 왔는지 확인하세요."
- Clear navigation context showing active state.

### Task 6: Consultation Page

**Output:** `app/consultation/page.tsx`  
**Requirements:** IA-02, UXR-02

- Client component.
- Move chat/citation/preference state from `StudentDashboard`:
  - `query`, `messages`, `isLoading`, `selectedCitation`, `sourceCitations`, `sourcePanelOpen`, `sourceOpener`
  - `submitQuestion`, `openCitation`, `closeSource`, `clearChatHistory`
- Render `ChatComposer`, `ChatMessageList`, `MobileSourceSheet`, `SourceInspectionRail`, `PreferencePanel`, `SettingsMenu`, `StorageScopeChip`, `DisclaimerNotice`.
- All existing component import paths unchanged.

### Task 7: CSS Migration

**Output:** Edit `app/globals.css`  
**Requirements:** UXR-02

- Keep CSS variables and shared component classes (card-surface, soft-surface, pill-control, primary-button, citation/listing/chat/preference classes).
- Rename Phase 5 shell classes:
  - `.phase5-shell` -> `.app-shell`
  - `.phase5-container` -> `.shell-container`
  - `.dashboard-header` -> `.shell-header` (or page-specific `.home-hero`)
  - `.dashboard-actions` -> `.shell-actions`
- Remove unused dashboard grid classes: `.dashboard-grid`, `.dashboard-left`, `.source-column`, `.panel-tabs`, `.mobile-panel`, old three-column rules.
- Add new shell/nav/route layout classes as needed.

### Task 8: Retire StudentDashboard

- Stop importing from `app/page.tsx`.
- Keep file if tests reference it; otherwise delete.
- All state/logic moved to Task 4 and Task 6.

## Acceptance Criteria

1. User can navigate between home, explore, source verification, and consultation through shared navigation.
2. User can see active state on the current page.
3. User does not see the crowded single-dashboard experience.
4. Navigation works on desktop and mobile.
5. All existing API routes and component interfaces unchanged.
6. No Phase 5 shell/grid class names remain in active pages.
