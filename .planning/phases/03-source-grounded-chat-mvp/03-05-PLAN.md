<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User query→ChatService | Untrusted Korean text enters orchestration and audit. |
| Provider output→ChatService | Model output is untrusted until validated. |
| Chat cycle→audit file | Sensitive metadata can be persisted if not bounded/redacted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Repudiation | `src/audit/audit-log.ts` | mitigate | Append one audit JSONL line for every chat cycle with trace ID, query hash, source IDs, prompt version, model safe config, guardrail results, and response timestamp. |
| T-03-05-02 | Information Disclosure | audit records | mitigate | Hash query by default, never log API keys/cookies/tokens, and store prompt snapshots only for refusal/guardrail/failure with reason. |
| T-03-05-03 | Tampering | audit file writes | mitigate | Use append-only JSONL via `appendFile` and schema-parse every record before writing. |
| T-03-05-04 | Elevation of Privilege | `src/chat/chat-service.ts` | mitigate | Validate provider output with `validateChatResponseOutput` and convert citationless/hostile outputs to guarded refusal. |
| T-03-05-05 | Denial of Service | hard-refusal path | mitigate | Skip provider calls for zero/weak evidence, reducing unnecessary external calls. |
</threat_model>
