<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Env config→provider request | API key and base URL enter runtime provider configuration. |
| Retrieved source text→prompt messages | Untrusted source text is included in model-visible context. |
| Provider response→validation | Provider output remains untrusted and is validated by Plan 03 and Plan 05. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Information Disclosure | `src/chat/openai-compatible-provider.ts` | mitigate | `getSafeConfig()` and thrown errors must never include `OPENAI_COMPAT_API_KEY`; tests use `secret-test-key`. |
| T-03-04-02 | Elevation of Privilege | `src/chat/prompt.ts` | mitigate | Place source text only inside `<retrieved_context source_text_trust="untrusted_source_text">` and state it cannot override safety, citation, privacy, or output-format instructions. |
| T-03-04-03 | Spoofing | prompt citations | mitigate | Build citation numbers and structured citation map from `RetrievedChunk.chunk.citation_anchors`, not model-supplied source labels. |
| T-03-04-04 | Denial of Service | provider request | mitigate | Support injected fetch and timeout/signal pattern; tests avoid live network calls. |
| T-03-04-05 | Tampering | prompt text | mitigate | Sanitize control characters from query and chunk text before prompt construction. |
</threat_model>
