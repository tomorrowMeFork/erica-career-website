<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| KB JSONL files→retrieval memory | Local artifacts are generated from public source text and remain untrusted evidence. |
| User request→chat contract | Korean user input enters the chat system and must be bounded before retrieval or prompts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `src/knowledge-base/jsonl-loader.ts` | mitigate | Parse every line with `KnowledgeChunkSchema`, require citation anchors, and reject any `source_text_trust` value other than `untrusted_source_text`. |
| T-03-01-02 | Denial of Service | `src/chat/chat-contract.ts` | mitigate | Bound `query` to 1..2000 characters and `top_k` to 1..10 in `ChatRequestSchema`. |
| T-03-01-03 | Information Disclosure | `src/chat/chat-contract.ts` | mitigate | Do not define any normal response field for full retrieved chunk text; expose only `answer`, `citations[]`, `refusal_tier`, `confidence`, and `trace_id`. |
| T-03-01-04 | Repudiation | loader errors | mitigate | Throw path-and-line-specific errors for invalid JSONL so failed KB inputs are diagnosable before chat execution. |
</threat_model>
