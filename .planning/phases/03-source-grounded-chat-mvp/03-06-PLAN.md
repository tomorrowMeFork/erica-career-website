## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args→chat query | User-supplied query enters local smoke/eval execution. |
| Eval mocks→quality gate | Mock responses can mask missing validation if cases are weak. |
| Env config→CLI output | Provider configuration errors can leak secrets if not redacted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06-01 | Spoofing | `scripts/evaluate-rag-mvp.ts` | mitigate | Eval asserts expected source/chunk in top 5 and structured citations rather than accepting any fluent answer. |
| T-03-06-02 | Tampering | hostile eval case | mitigate | Include hostile-source injection case attempting to suppress citations and verify validation rejects it. |
| T-03-06-03 | Repudiation | eval output | mitigate | Print deterministic failures and `rag mvp evaluation passed` only after all checks pass. |
| T-03-06-04 | Information Disclosure | `scripts/chat-smoke.ts` | mitigate | Print only structured response fields and redact provider errors; never print API key, full prompt, or full chunk text. |
| T-03-06-05 | Denial of Service | eval default path | mitigate | Use deterministic mock provider by default and avoid live LLM calls unless optional env config exists. |
