# Plan 07-01: UI Information Architecture and Design Contract

**Phase:** 07 — UI Information Architecture and Design Contract  
**Status:** Ready for execution  
**Requirements:** IA-01, IA-03, UXR-01, UXR-03  
**Created:** 2026-05-04

## Goal

Create the v1.1 UI information architecture and design contract so Phase 8+ can implement a clear four-page experience without repeating the failed Phase 5 dashboard direction.

## Scope

This plan creates planning/design artifacts only. It does not implement routes, components, CSS, APIs, retrieval, ingestion, ranking, matching, saved jobs, reminders, application tracking, SSO, or official endorsement claims.

## Task Breakdown

### Task 1: Page IA Contract

**Output:** `07-01-PAGE-IA-CONTRACT.md`  
**Requirements:** IA-01, IA-03

Define the four-page model:

- home
- career information explore
- information detail/source verification
- career consultation

For each page, specify purpose, user job, primary action, secondary actions, empty state, evidence/source behavior, and prohibited implication.

### Task 2: Flow Language Contract

**Output:** `07-02-FLOW-LANGUAGE-CONTRACT.md`  
**Requirements:** IA-01, IA-03

Define Korean-first product labels, navigation labels, CTA labels, helper copy, scope-limit copy, and a framing audit that replaces job-board or recommendation-system language with source-grounded consultation language.

### Task 3: Design Standard Interpretation

**Output:** `07-03-DESIGN-STANDARD-INTERPRETATION.md`  
**Requirements:** UXR-01, UXR-03

Translate `DESIGN.md` into ERICA Career Chat rules for spacing, typography, rounded surfaces, pill controls, restrained color roles, CTA hierarchy, source cards, and page-level visual priority.

This task must include a Phase 5 anti-pattern section stating that Phase 5 is useful only for artifact format and historical critique, not target design direction.

### Task 4: Human Intervention Register

**Output:** `07-04-HUMAN-INTERVENTION-REGISTER.md`  
**Requirements:** IA-01, IA-03, UXR-01, UXR-03

Record decisions that may need later human input. For each item, include recommended conservative default, reason, risk if wrong, and the phase where it should be revisited.

### Task 5: Verification Acceptance Checks

**Output:** `07-05-VERIFICATION-ACCEPTANCE-CHECKS.md`  
**Requirements:** IA-01, IA-03, UXR-01, UXR-03

Define binary pre-implementation acceptance checks for purpose clarity, exploration-vs-consultation distinction, primary CTA hierarchy, Korean-first copy, source-grounding limits, design-standard conformance, and scope guardrails.

### Task 6: Summary

**Output:** `07-SUMMARY.md`

Summarize completed artifacts, requirement coverage, deferred items, and readiness for Phase 8.

## Acceptance Criteria

- IA-01 is covered by home purpose, source-grounded model, primary actions, and service-limit copy.
- IA-03 is covered by separate exploration and consultation page contracts, labels, and CTA hierarchy.
- UXR-01 is covered by project-specific interpretation of `DESIGN.md`.
- UXR-03 is covered by single-primary-action rules and restrained color/badge/card/CTA hierarchy.
- Human-intervention items are recorded separately with recommended defaults.
- No artifact proposes implementation or excluded v1.1 scope.
- Phase 5 is explicitly treated as an anti-pattern for UI direction.
