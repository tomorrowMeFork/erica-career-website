## Typography

Use exactly four size roles and two weights. Do not use extra font weights in Phase 05 UI.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.7 | Korean chat answers, listing descriptions, notice bodies |
| Label | 14px | 400 | 1.5 | Metadata, filters, helper text, source dates, privacy copy |
| Heading | 20px | 600 | 1.35 | Section headers, card titles, answer titles, panel titles |
| Display | 28px | 600 | 1.25 | Dashboard title and primary empty-state heading only |

Additional typography rules:

- Weights allowed: `400` regular and `600` semibold only.
- Korean body copy: `letter-spacing: 0`; do not apply Meta/Optimistic negative tracking.
- Dense source metadata may use 14px/1.5 but must remain readable and never drop below 12px for captions.
- Inline citation markers `[1]` use 14px/600, Hanyang-blue text, underline on focus, and `aria-label="1번 출처 보기"`.
- Long Korean titles use two-line clamp on cards only; chat answers and source panel titles must not truncate critical context.

## Breakpoints and Responsive Contract

| Breakpoint | Width | Layout Contract |
|------------|-------|-----------------|
| Mobile | `<768px` | Single-column dashboard. Chat is primary. Listings/preferences/source inspection open as bottom drawer/sheet. Composer is sticky bottom within safe-area inset. |
| Tablet | `768px–1023px` | Chat remains central. Listings/preferences may appear as collapsible panels above/below chat. Source inspection uses full-height drawer or modal sheet. |
| Desktop | `1024px–1359px` | Three-zone dashboard: optional left preference/listing rail `280px`, central chat flexible, right source rail `360px` when citation selected. |
| Wide | `>=1360px` | Same as desktop with max container `1440px`; right source rail may grow to `400px`. |

Responsive rules:

- Opening a source on desktop must not navigate away from chat; it opens/updates the right rail.
- Opening a source on mobile uses a bottom sheet/drawer with focus trapped inside, close button first in tab order, and previous focus restored on close.
- Listing browse is outside the chat flow but within the single dashboard via a tab/panel, not a separate primary page.
- Preference controls are side panel on desktop and drawer/bottom sheet on mobile.
- Chat history remains in client session state only; refreshing may clear unless future persistence is explicitly consented.

## Interaction Contracts

### Complete-Response Chat Flow

1. User enters Korean query and activates `질문 보내기`.
2. UI appends a pending user/assistant pair to client-session history.
3. UI displays skeleton/notice copy: `관련 출처를 확인하고 답변을 준비하고 있어요…`.
4. Thin route handler returns a complete `ChatResponse`; no streaming UI in Phase 05.
5. UI validates/render-guards shape: answer, citations, refusal tier, confidence, trace ID.
6. If `refusal_tier` is `hard_refuse`, show refusal notice and no unsupported answer expansion. If citations exist, still allow source inspection.
7. If `soft_hedge`, show answer plus `확인 필요` notice and citation trigger.
8. Every cited factual answer must expose citation markers and `N개 출처 사용` trigger.

### Source Inspection

- Source inspection opens in-place: right rail on desktop, bottom sheet/drawer on mobile.
- Source card fields are required: title, URL, fetched date, posted date when non-null, deadline badge, page number when present.
- Official URL opens in new tab with `rel="noopener noreferrer"` and accessible label `공식 페이지 새 창으로 열기`.
- Selected source remains visually highlighted with accent border and not only color.
- Closing rail/sheet returns focus to the trigger that opened it.

### Listing Browse

- Default view: recommended-first cards. When no preferences exist, label mode as `기본 추천` and order by latest/active/source-grounded service result.
- Filters are client UI controls over available response categories/statuses where possible; server refetch is allowed but must preserve selected filter and loading state.
- `마감됨` listings are visibly muted but still readable and openable for verification.
- `마감일 확인 필요` listings show amber warning and encourage official-page verification.
- No advanced text search, saved jobs, reminders, application flow, or resume tools in Phase 05.

### Preference Controls

- Required fields `major` and `target_role` are shown first.
- Optional fields are hidden under an accordion labeled `선택 조건 더보기`.
- Session-first storage is the default. Persistent storage may appear only behind explicit opt-in with consent timestamp, retention days, and deletion support.
- Clearing preferences disables preference-based ranking and updates visible storage scope.
- Preference UI must never imply inferred profiling; copy uses `입력한 조건` not `분석한 관심사`.

## Accessibility Requirements

- Meet WCAG AA contrast for text, icons, focus rings, badges, and disabled states.
- All interactive controls have visible `:focus-visible` ring: `2px solid accent` plus 2px offset, or equivalent non-color outline.
- Minimum touch target: 44px by 44px for buttons, filters, source triggers, sheet close buttons, settings controls, and listing CTAs.
- Keyboard support:
  - Tab order follows dashboard header, panel controls, chat history actions, composer, side panels.
  - `Esc` closes source rail/sheet, settings menu, confirmation dialog, and mobile drawer.
  - Arrow keys may move within tab/filter groups only if ARIA roles are implemented correctly; otherwise use standard buttons.
- Dialog/sheet behavior:
  - Mobile source/preference sheets trap focus while open.
  - Initial focus lands on sheet title or close button.
  - Focus returns to invoking citation/filter/settings trigger on close.
  - Background content is inert or `aria-hidden` while modal drawer is active.
- Screen reader labels:
  - Citation marker: `1번 출처 보기`.
  - Source link: `{title} 공식 페이지 새 창으로 열기`.
  - Deadline badge: `마감 상태: 모집중|마감됨|마감일 확인 필요`.
  - Trace ID details: collapsed by default with `응답 추적 ID 보기`.
- Korean language:
  - Set `<html lang="ko">`.
  - Use Korean labels before English technical identifiers.
  - Preserve citation IDs and trace IDs exactly as machine-readable strings.

## Implementation Non-Goals and Deferred Items

- Streaming chat responses.
- Server-side chat-history persistence.
- Advanced listing search/filtering beyond core pill filters.
- Official Hanyang SSO or claims of official endorsement.
- Resume upload, cover-letter tools, interview prep tools, application submission, saved jobs, or reminders.
- Production crawling, authenticated/private source crawling, or bypassing source access controls.
- Dark mode, custom brand illustration system, product-gallery/SKU/checkout patterns, Meta branding, Optimistic VF, Oculus purple, or commerce language.
- Full Phase 6 release/evaluation suite, though Phase 6 safety copy placeholders must not be contradicted.
