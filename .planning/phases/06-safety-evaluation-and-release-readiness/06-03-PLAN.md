<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| QA dataset→ChatService/retriever | Local fixtures drive evaluation through product service boundaries. |
| optional env names→judge provider | Optional live judging is enabled only by env variable names and must not leak values. |
| eval report→operator | Release output guides go/no-go decisions and must be safe and source-faithful. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Tampering | retrieval evaluator | mitigate | Use `Bm25Retriever` + `loadKnowledgeBaseChunks`; assert expected top results and metadata rather than hand-rolled retrieval. |
| T-06-03-02 | Information Disclosure | eval CLI reports | mitigate | Safe result shape excludes raw prompts, raw source text, `.env` values, API keys, and provider payloads; tests assert redaction. |
| T-06-03-03 | Spoofing | answer evaluator | mitigate | Reject official endorsement and guaranteed outcome phrases through deterministic checks and optional judge dimensions. |
| T-06-03-04 | Tampering | hostile source fixture | mitigate | Treat hostile text as untrusted retrieved content; assert citations remain and hostile instructions/unsafe claims are absent. |
| T-06-03-05 | Denial of Service | default eval command | mitigate | Default path uses deterministic local fixtures and does not depend on missing provider credentials, live network, or production crawling. |
</threat_model>
