## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local files→status reader | Local manifests/JSONL may be missing, stale, malformed, or untrusted. |
| status reader→operator CLI | Summary informs release readiness and must not expose sensitive data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Tampering | manifest parsing | mitigate | Validate manifests with Zod and report invalid/missing artifacts as warnings/failures without guessing. |
| T-06-04-02 | Elevation of Privilege | freshness CLI | mitigate | CLI reads local files only and never triggers crawling, auth, Playwright, or network fetches per D-21/D-25. |
| T-06-04-03 | Information Disclosure | status output | mitigate | Output aggregate metadata only: source IDs, counts, timestamps, warnings; exclude raw source text, prompts, `.env`, keys, and stack traces. |
| T-06-04-04 | Repudiation | release freshness evidence | mitigate | Include evaluated-at timestamp and per-source status so checklist/final gate can record what was checked. |
</threat_model>
