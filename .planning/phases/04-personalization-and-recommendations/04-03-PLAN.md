<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source candidates → match reasons | Untrusted source content may influence user-facing explanations. |
| service response → future UI/chat | Recommendation output becomes user-facing Korean employment guidance. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | `buildMatchReasons` | mitigate | Use fixed templates and allowed citation IDs; never obey source text instructions. |
| T-04-03-02 | Spoofing | match reason wording | mitigate | Reject official endorsement and guaranteed outcome phrases. |
| T-04-03-03 | Information Disclosure | `RecommendationService` output | mitigate | Omit raw source text and session-only optional preference text from response schemas. |
| T-04-03-04 | Repudiation | recommendation response | mitigate | Include `trace_id`, structured citations, score breakdowns, and privacy metadata. |
</threat_model>
