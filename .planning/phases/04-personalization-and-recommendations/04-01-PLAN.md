## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| student input → preference schemas | Untrusted preference text enters runtime validation. |
| preference service → store | Personal data crosses into session or durable storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Information Disclosure | `PersistentPreferenceProfileSchema` | mitigate | Reject `session_only_optional_text`, unknown keys, inferred attributes, and unnecessary identifiers before persistent writes. |
| T-04-01-02 | Repudiation | `writePersistent` | mitigate | Require `PreferenceConsentSchema` with `consented_at`, `retention_days`, and `deletion_supported: true`. |
| T-04-01-03 | Elevation of Privilege | `InMemoryPreferenceStore` session keys | mitigate | Scope read/write/clear by explicit `sessionKey`; test that clearing one key does not affect another. |
| T-04-01-04 | Tampering | preference lifecycle input | mitigate | Parse every set/update path through strict Zod schemas before storing. |
</threat_model>
