# Phase 2 Plan 02-06: Complete `sources.txt` Authenticated Collection Coverage

## Goal

Make every source intent in `sources.txt` collectable or explicitly blocked by current, observed evidence. This plan reopens Phase 2 because fixture-first sample ingestion did not satisfy the full source-coverage goal.

## Scope

- `https://cdp.hanyang.ac.kr/` root and observed same-host navigation surface.
- CDP `취업정보 > 하위항목 전체`.
- CDP `채용정보 > 하위항목 전체`.
- `https://book.hanyang.ac.kr/Viewer/YKCF2I67RO4B` 취업성공후기 viewer.
- CDP guidebook PDF.
- `https://ibus.hanyang.ac.kr/front/recruit/r-1` 경상대학 취업정보 board.

## Requirements

- Load local `.env` credentials without printing values.
- Keep `.env`, cookies, Playwright auth state, traces, screenshots, and generated raw/live corpora out of git.
- Use one browser context/page, same-host routing, `COLLECT_MAX_PAGES`, and `COLLECT_DELAY_MS` for bounded low-concurrency discovery.
- Preserve Korean source labels, official URLs, fetched timestamps, citation anchors, deadline status where available, and `source_text_trust` markers.
- Keep `scheduled_crawling_enabled: false` for all sources.

## Tasks

1. Add `.env` loading and sanitized credential-presence reporting to CDP discovery.
2. Update CDP seed discovery to follow the observable same-host public surface at `/Main/default.aspx` when the root body is empty.
3. Record observed CDP `취업정보` and `채용정보` candidate URLs from the current run.
4. Implement bounded CDP listing/detail parsers only for observed URL shapes.
5. Implement book viewer discovery to identify whether original download, page assets, or DOM text can be cited safely.
6. Extend ibus collection from single fixture/sample to bounded page/detail collection with the configured page limit.
7. Write JSONL KB outputs for each collectable source and verify source/citation/freshness invariants.
8. Update source registry, access evidence, roadmap, and state based on observed results.
