<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator-edited YAML → validator | Untrusted local registry text is parsed and validated before downstream ingestion can use it. |
| Source evidence text → planning artifacts | robots/ToS evidence may contain unexpected text and must not become executable logic or authorization by itself. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Source registry YAML | mitigate | Validate every record with Zod enums/literals; reject duplicate `source_id`; require exact fields from D-13. |
| T-01-02 | Repudiation | Authorization metadata | mitigate | Keep `approval_basis`, `approval_status`, `robots_status`, and `tos_status` as separate required fields per D-07/D-08. |
| T-01-03 | Elevation of Privilege | Scheduled crawling flag | mitigate | Require `scheduled_crawling_enabled: false` with `z.literal(false)` until access review is complete per SAFE-05. |
| T-01-04 | Information Disclosure | Validator errors/logging | mitigate | Do not read `.env`, credentials, cookies, or Playwright storage in this plan; validator reports schema paths only. |
</threat_model>
